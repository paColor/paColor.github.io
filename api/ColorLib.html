CFForceBlack

CharFormatting contient un certain nombre de méthodes qui permettent de savoir comment se comporter avec une valeur false pour les différents membres de type bool. Faut-il ne rien faire ou forcer la caractéristique "gras" à "non gras" par exemple?

Une instance de CFForceBlack se comporte comme un CharFormatting mais retourne toutjours true pour les méthodes évoquées ci-dessus. Cela permet par exemple de créer un CharFormatting qui forcera la mise en noir et l'élimination des caractéristique bold, italic, underline et autres.

ColConfWin

Classe pour la gestion des éléments de configuration nécessaires pour la colorisation des phonèmes. C'est ici qu'on trouve la définition des "sons" qui sont utilisés par l'interface utilisateur. On trouve en particulier le 'mapping' entre les sons et les Phonemes. La table correspondante (sonMap) est de fait la définition formelle des sons et de leur nom, tels qu'ils sont acceptés par le programme.

Pour chaque son on a un flag (checkbox) qui indique si la colorisation pour ce son est activée et un CharFormatting à utiliser si le flag correspondant est mis. Les méthodes principales permettent de lire et d'écrire ces deux paramètres pour chaque son.

TheText

The class TheText represents the text that is analysed and whose formatting is adapted by the analysis algorithm.

The idea is that the inheriting class is created with a string. Once this is done, different methods make it possible to apply some kind of formatting to the text. This formatting is applied to the inheriting class by calling the method SetChars see SetChars(FormattedTextEl, Config).

UnsetBehConf

UnsetBehaviourConfiguration:

Configuration pour les flags avancés déterminant comment doit se comporter le formatage quand un attribut d'un CharFormatting est à la valeur false. Faut-il laisser le formatage dans l'état actuel ou forcer à la valeur false?

Chacun des flags indique comment traiter un des attributs de CharFormatting. La liste des attributs gérés par la classe est donnée par l'enum Ucbx. On peut également accéder à chaque flag par son nom sous forme de string. Les valeurs admises sont: 

{ "Bold",       Ucbx.bold },
{ "Italic",     Ucbx.},
{ "Underline",  Ucbx.underline },
{ "Color",      Ucbx.color },
{ "Hilight",    Ucbx.hilight },
{ "All",        Ucbx.all }
